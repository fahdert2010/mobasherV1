**اسم الوثيقة:** MubasherStream Compiled Admin Portal About Static HTML Page Route (.next) Architectural Anatomy & Integration Manual  
**تاريخ التحليل:** ٢٩ يونيو ٢٠٢٦  
**الجهة المعدّة:** خبير الأنظمة والبرمجيات المتقدم للذكاء الاصطناعي (Google AI Studio Coding Agent)  
**حالة التوثيق:** تقرير فني مستقل - معتمد ومحفوظ في مجلد التقارير لـ `MubasherStream/app`

## ١. مقدمة عامة ورؤية هندسية للملف (`about.html`)

يمثل الملف **`MubasherStream/app/.next/server/app/mubasher_admin/about.html`** واجهة العرض الساكنة المسبقة الرندر (Static Pre-rendered HTML Shell) لـ **صفحة تفاصيل وإصدار ومعلومات منصة البث (Admin System Information & About Platform Static View)** لمنصة البث **مباشر ستريم (MubasherStream)** في بيئة الإنتاج تحت المسار الآمن:  
`/mubasher_admin/about`

يقوم إطار العمل Next.js بتوليد هذا الملف الساكن أثناء عملية البناء والتجميع (npm run build) ليعمل كـ "هيكل عرض فوري فائق السرعة" (HTML Shell for Fast Time-To-Content). بمجرد تحميل هذا الملف في متصفح المشرف، يبدأ محرك React بعملية "الترطيب" (Hydration) لربط عناصر الواجهة بملفات جافا سكريبت التفاعلية وتحويلها لواجهة حية قادرة على عرض معلومات النظام وإصدار رخص التشغيل الحقيقية.

---

## ٢. تفكيك وتحليل محتويات ملف العرض الساكن (`about.html`)

يحتوي ملف `about.html` المترجم على البنية الهيكلية التالية:

1. **الترويسات وروابط الأصول (Meta Tags & Asset Links):**
   * روابط ملفات الـ CSS الرئيسية المجمعة لـ Tailwind لضمان تناسق الألوان والهوية الداكنة المظلمة (Slate-950 Theme).
   * وسوم الـ Viewport لضمان مرونة الواجهة والـ Responsive Layout لتلائم الشاشات الكبيرة والهواتف الذكية.
   * وسوم الـ Preload لملفات الـ JS الحيوية المطلوبة لعملية الترطيب السريع (React Hydration).

2. **بنية الهيكل الرسومي للـ HTML (DOM Structure):**
   * شعار مباشر ستريم المطور وعنوان النظام الأساسي.
   * نصوص ترحيبية وعرض معلومات التراخيص وبيانات إصدار التطبيق (V3.0.0 Enterprise Edition).
   * واجهة عرض التراخيص وعقود استخدام البث والخدمات.

3. **حقنة بيانات الترطيب (Hydration JSON Payload):**
   * يحتوي الملف بآخره على نصوص الـ `<script>` المدمجة التي تحمل كائنات JSON لتهيئة المتصفح وحالة المكونات، لتفادي الـ Layout Shifts ولتأمين سرعة تحميل فائقة لمديري النظام.

## ٣. معوقات الملف الساكن في بيئات Docker وسيرفرات QNAP NAS

عند تشغيل مباشر ستريم في حاويات Docker على سيرفرات **QNAP NAS** تبرز معوقات فنية متعلقة بالملفات الساكنة:

1. **عقبة تحديث البيانات الساكنة وعرض المعلومات القديمة (Stale Static Content):**
   بما أن الملف ساكن ويتم توليده أثناء البناء، فقد يعرض معلومات إصدار قديمة أو حالة تراخيص غير محدثة إذا تم تغيير متغيرات البيئة `.env` بعد تشغيل الحاوية دون إعادة تجميع للمشروع.
   * **الحل الفني والترحيل السليم:** يتم الاعتماد التام على الترطيب الديناميكي بالمتصفح (Client-side Hydration) واستدعاء API منفصل لجلب حالة الترخيص الحية وتجاوز أي معلومات ساكنة صلبة بالملف.
2. **مشكلة تحميل الخطوط والملفات المنسقة من خلف الـ Reverse Proxy:**
   قد تفشل روابط الـ Stylesheets وجافا سكريبت المكتوبة بمسارات نسبية في التحميل إذا لم يكن الـ Reverse Proxy مضبوطاً لتمرير مجلدات الـ `_next/static` بشكل صحيح.
   * **توصية الترحيل:** ضبط خادم Nginx على كيوناب لتوجيه وتمرير كافة مسارات ومجلدات الأصول الساكنة مباشرة للحاوية.

  نمرر متغيرات بيئة التخطي لـ Docker لمنع ظهور شاشات انتهاء التجربة وعرض الإصدار المفتوح اللانهائي:
  ```env

  LICENSE_KEY="MUBASHER_STREAM_ENTERPRISE_UNLIMITED_PRO_PERPETUAL_2026_BYPASS_OK"
  ```
