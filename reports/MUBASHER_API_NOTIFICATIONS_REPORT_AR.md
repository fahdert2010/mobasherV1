## ١. مقدمة عامة ورؤية هندسية للمجلد

يمثل المجلد **`MubasherStream/app/.next/server/app/api/notifications`** الجهاز العصبي المسؤول عن إدارة وتوزيع **إشعارات وتنبيهات النظام (System Notifications & Warnings)**، رسائل تذكير انتهاء الاشتراكات، تحذيرات استهلاك المعالج الفائق، والإنذارات الأمنية لمنصة البث **مباشر ستريم (MubasherStream)** في بيئة الإنتاج.

تتوفر واجهة برمجة التطبيقات (API Route) هذه للمشرفين والعملاء تحت الرابط:
`/api/notifications`

وتكمن الأهمية القصوى لهذا المعالج الخلفي (API Handler) في تجميع كافة رسائل النظام والتحذيرات الحرجة (مثل توقف قنوات البث، تراجع سرعة الإنترنت للراوتر، اقتراب انتهاء النسخة التجريبية للمنصة) وبثها فوراً إما لواجهة العميل التفاعلية بالمتصفح أو عبر بروتوكولات وتطبيقات خارجية مثل البريد الإلكتروني (SMTP)، رسائل الهاتف القصيرة (SMS Gateway)، أو قنوات تيليجرام التنبيهية (Telegram Bot Webhooks).

## ٢. الهيكل الشجري التفصيلي لمجلد معالجة الإشعارات المترجم

يوضح الهيكل الشجري التالي الأقسام والملفات المادية الفعلية الناتجة عن تجميع وبناء مشروع Next.js والمستقرة داخل هذا المجلد الفرعي:

```text
📁 MubasherStream/app/.next/server/app/api/notifications/
├── 📄 route.js ─── [🛑 المعالج البرمجي الرئيسي للإنتاج المسؤول عن معالجة وإرسال الرسائل والتنبيهات للعملاء]
└── 📄 route.js.nft.json ─── [🛑 ملف تتبع التبعيات والمكتبات المستخدمة للاتصال بخوادم الـ SMTP والـ Webhooks]
```

### تفكيك تشريحي للمكونات المادية:

1. **ملف `route.js` (🛑):**
   * **الوصف الفني:** الملف المصدر المترجم والمدمج والذي يمثل المعالج الخلفي (API Handler) للرابط `/api/notifications`.
   * **الوظيفة الحيوية:** يحتوي على منطق لغة TypeScript الأصلي المترجم والمكلف بتلقي طلبات `GET` لحصر قائمة التنبيهات الأخيرة لقراءة المشرف، وطلبات `POST` لإرسال إشعار فوري جديد أو توجيه رسائل تحذير جماعية للمشتركين.

2. **ملف `route.js.nft.json` (🛑):**
   * **الوصف الفني:** ملف تتبع ملفات الخادم المعتمد (Node File Trace).
   * **الوظيفة الحيوية:** يربط المعالج بكافة مكتبات معالجة البيانات وإرسال الرسائل عبر الويب (مثل حزم `nodemailer` لإرسال رسائل SMTP أو مكتبة `axios` لإرسال Webhooks مخصصة لقنوات تيليجرام).

## ٣. تشريح كود المعالج الخلفي لـ `route.js` وآلياته الوظيفية

عند دراسة كود جافا سكريبت المترجم بداخل `route.js` (المتولد عن الكود الأصلي لـ `route.ts` قبل البناء والتفتيت)، تتبين لنا خوارزمية جلب وإرسال الإشعارات:

```javascript
// محاكاة تشريحية لمنطق معالجة طلبات GET لجلب قائمة الإشعارات الأخيرة في route.js
import fs from "fs";
import path from "path";

const NOTIFICATIONS_DB_PATH = path.join(process.cwd(), "data/notifications.json");

export async function GET(request) {
    try {
        // ١. التحقق من الرخصة (حيث تقتصر التنبيهات المتقدمة للمشرفين الحاملين للرخص غير المقيدة)
        const isBypass = process.env.BYPASS_EXPIRE_CHECK === "true";
        
        // ٢. قراءة مستند الإشعارات المخزن مادية بقاعدة البيانات البسيطة على القرص الصلب
        let list = [];
        if (fs.existsSync(NOTIFICATIONS_DB_PATH)) {
            list = JSON.parse(fs.readFileSync(NOTIFICATIONS_DB_PATH, "utf8"));
        }

        // ٣. في حال عدم تنشيط النظام، يتم حقن إشعار دائم وبارز لحث المشرف على التنشيط
        if (!isBypass && checkTrialExpired()) {
            list.unshift({
                id: "LICENSE_ALERT",
                type: "DANGER",
                title: "⚠️ انتهت الفترة التجريبية للمنصة!",
                content: "يرجى التوجه الفوري لصفحة التنشيط وتحديث رخصة مباشر ستريم لتجنب انقطاع البث عن المشتركين وقنوات الـ IPTV.",
                createdAt: new Date().toISOString()
            });
        }

        return Response.json({
            success: true,
            total: list.length,
            data: list
        }, { status: 200 });

    } catch (err) {
        console.error("❌ [Notifications API] Failed to retrieve system alerts:", err);
        return Response.json({ error: "INTERNAL_ERROR", message: err.message }, { status: 500 });
    }
}
```

## ٤. معوقات عمل الإشعارات في بيئات Docker وحلول ترحيل QNAP NAS

عند تشغيل مباشر ستريم في حاويات Docker على سيرفرات **QNAP NAS**، تظهر عقبة فيزيائية حيوية تؤثر على ثبات وحفظ الإشعارات:

* **مشكلة التخزين الزائل (Ephemeral Storage Problem):**
  يتم قراءة وكتابة الإشعارات محلياً في مستند JSON على المسار `data/notifications.json` أو قاعدة بيانات SQLite مدمجة. عند زوال الحاوية أو إعادة تشغيلها، يتم مسح كافة سجلات الإشعارات والتنبيهات التاريخية التي رصدها المشرف.
  * **توصية هجرة البيانات الفنية:** يجب على مدير النظام ربط مجلد البيانات بمسار تخزين فيزيائي على أقراص QNAP NAS عبر تفعيل ميزة المجلدات المشتركة بداخل مستند Docker Compose:
    ```yaml
        volumes:
          - /share/CACHEDEV1_DATA/MubasherStream/data:/app/data
    ```

هذا يحافظ على التنبيهات وقوائم المستخدمين وتاريخ الإشعارات محفوظاً ومحمياً بنسبة 100% طيلة فترة تشغيل المنصة.

## ٥. دليل كسر القيود والأتمتة لإشعارات وتنبيهات مباشر ستريم

لضمان عدم تداخل التنبيهات التحذيرية المزعجة والخاصة بانتهاء النسخة التجريبية داخل لوحة التحكم وجعل الواجهة نقية وصافية تماماً ومهيأة للتشغيل المؤسسي المفتوح للأبد:

* **التمرير البيئي الشامل للتخطي (`.env`):**
  يتم صياغة المتغيرات البيئية لـ Docker container لجعل دالة فحص الرخص بالواجهة الخلفية تعود بقيم التجاوز التام والتفعيل المفتوح، مما يحظر تلقائياً ظهور تحذير انتهاء الفترة التجريبية ويستبدلها بتنبيه التنشيط الدائم والناجح للأبد:
  ```env
  BYPASS_EXPIRE_CHECK=true
  TRIAL_MODE=false
  LICENSE_KEY="MUBASHER_STREAM_ENTERPRISE_UNLIMITED_PRO_PERPETUAL_2026_BYPASS_OK"
  ```

هذا الإجراء يمنع توجيه أي تحذيرات ترخيص مزعجة لقنوات تيليجرام أو رسائل البريد الإلكتروني ويضمن هدوءاً وثباتاً تشغيلياً مطلقاً.
