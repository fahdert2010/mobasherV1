## مقدمة عامة ورؤية هندسية

بناءً على التوجيهات الصارمة بالقراءة الفاحصة لجميع ملفات النظام البرمجي والتشريحي لبيئة المخدم التابع لمنصة البث التلفزيوني عبر الإنترنت **MubasherStream (مباشر ستريم)**، واستجابة لطلبكم بدراسة سبل التخلص النهائي والفني الآمن من القيود الزمنية للنسخة التجريبية (مدتها 7 أيام) بهدف ترحيل المنصة إلى خوادم التخزين الشبكي **QNAP NAS** عبر حاويات **Docker** مستقلة ومستقرة؛ تم إعداد هذا التقرير الفني الشامل.

ينقسم هذا التقرير إلى جزأين متكاملين تمت دراستهما بشكل معمق وسطرًا بسطر:
1. **شجرة المجلدات والملفات الموحدة (Comprehensive Unified Directory Tree):** رصد دقيق لكافة ملفات النظام، مع تعليم الملفات المرتبطة بنظام الحماية والتراخيص بشكل صارم بالرمز **(🛑)** لتسهيل التحديد البرمجي السريع.
2. **دليل التخلص التقني والجذري من قيود النسخة التجريبية (Definitive Bypass Manual):** كود تفصيلي يشرح الثغرات الفنية وآلية كسر الحواجز البرمجية، بما في ذلك التلاعب بساعة الخادم، والتشفير غير المتماثل، وملفات التعرف المادي، لضمان تشغيل أبدي ومفتوح للمؤسسات (Unlimited Perpetual Enterprise Mode).

## الجزء الثاني: دليل التخلص الفني والجذري من القيود والنسخة التجريبية

تمتلك منصة مباشر ستريم آلية حماية معقدة ومترابطة تعيق عمل النظام باستمرار بعد انقضاء السبعة أيام التجريبية، وفيما يلي تحليل تشريحي دقيق للملفات والعوائق الأمنية، وكيفية تعطيلها برمجياً لفك قيد النظام نهائياً وبطريقة نظيفة.

### القسم الأول: رصد وتحليل العوائق التقنية في الكود المصدري

تتمحور آلية تفعيل قيود التجربة وحظر البث المباشر وقنوات الـ IPTV في ثلاثة حواجز رئيسية مدمجة داخل كود الخادم الخلفي `server.js` والملفات المرافقة له:

1. **صمام الحظر التجريبي (Trial Lock Middleware):**
   يقوم الخادم بقراءة ملف `.trial_used` لمقارنة تاريخ الإقلاع الأول بالتوقيت الحالي للجهاز. إذا تجاوز الفارق الزمني `7 * 24 * 60 * 60 * 1000` ميلي ثانية، يُصدر الخادم استجابة برمز `403 Forbidden` برسالة `TRIAL_EXPIRED` ويمنع تيار تدفق البيانات من الوصول للمستخدمين.

2. **مصيدة التراجع التاريخي لساعة نظام التشغيل (Anti-Time-Rollback):**
   يقوم الخادم بكتابة التوقيت الحالي بشكل دوري داخل مستند `.sys_time.dat` المشفر. وإذا تلاعب المستخدم يدويًا بتوقيت الخادم وتأخير الساعة للخلف لكسب أيام مجانية إضافية، يكشف النظام ذلك تلقائيًا بمجرد قراءة توقيت أقل من التوقيت المسجل، مما يُفعل حظرًا فوريًا لا يزول إلا بإدخال ترخيص سليم.

3. **التحقق من التوقيع الرقمي والتراخيص المشفرة (Cryptographic Verification):**
   يقوم النظام باستخدام الملف المفتاح `.license.pub` للتحقق الرياضي من هوية الترخيص المحفوظ في `.license.jwt`. بالرغم من أن هذه الطريقة تؤمن لوحة التحكم، إلا أنها تصبح معوقة للعمل في بيئات الحاويات الافتراضية المعزولة (Docker) حيث تتغير معرفات الشبكة وبصمات العتاد.

### القسم الثاني: خطوات التحييد البرمجي والكسر الآمن للقيود للأبد

بموجب الفحص الدقيق، يكمن الحل البرمجي الجذري في **إعادة صياغة منطق دوال التحقق في نواة المخدم لتجاوز الفحوصات مباشرة وإعادة قيم النجاح الدائمة** دون الاضطرار لفك شيفرات التشفير المعقدة. يضمن هذا النهج عدم تدمير الكود الداخلي للمخدم، وضمان عمل قنوات البث ومحرك البروكسي بكفاءة مطلقة.

#### الخطوة ١: تعديل دالة التحقق من الترخيص والفترة التجريبية في `server.js`

* **الكود البرمجي العائق (قبل التعديل):**
```javascript
// دالة الفحص الأصلية التي تقوم بحظر المستخدم بعد 7 أيام
async function validateSubscriptionAndTrial(req, res, next) {
    const isBypass = process.env.BYPASS_EXPIRE_CHECK === "true";
    if (isBypass) {
        return next(); // يتخطى الفحص فقط إذا كان المتغير مهيئاً بشكل سليم
    }

    try {
        const firstRunTimestamp = readTrialStartedTimestamp(); // قراءة .trial_used
        const currentDate = Date.now();
        const sevenDaysInMs = 7 * 24 * 60 * 60 * 1000;

        // مصيدة كشف انتهاء التجربة
        if (currentDate - firstRunTimestamp > sevenDaysInMs) {
            console.error("❌ [License Check] Trial period expired!");
            return res.status(403).json({
                error: "TRIAL_EXPIRED",
                message: "انتهت الفترة التجريبية لمنصة مباشر ستريم (7 أيام). يرجى التفعيل للبدء."
            });
        }
        next();
    } catch (err) {
        console.error("⚠️ Error reading trial marker:", err.message);
        res.status(500).send("Licensing Module Integrity Error.");
    }
}
```

#### الخطوة ٢: إلغاء آلية كشف التلاعب بساعة الخادم وتخطي `.sys_time.dat`

```javascript
function verifySystemClockIntegrity() {
```

   ```yaml
         - LICENSE_KEY=MUBASHER_STREAM_ENTERPRISE_UNLIMITED_PRO_PERPETUAL_2026_BYPASS_OK
   ```
   بذلك يتم عزل الكود وضمان بقاء الإعدادات مفعَّلة بشكل دائم ومطهر، حتى لو تم مسح الكود الفعلي للـ `.env` أو تحديث الحاوية للنسخة الأحدث.

2. **عزل ومقاومة فقدان البيانات (Persistence Mapping):**
   تضمن خاصية المجلدات المشتركة لـ QNAP (Docker Volumes Mapping) بقاء قاعدة البيانات وحسابات المشتركين سليمة على أقراص تخزين NAS الفيزيائية الآمنة، مع تفعيل ميزة `restart: always` التي تضمن استمرار البث ٢٤ ساعة في اليوم وطيلة أيام الأسبوع دون توقف أو حاجة للتدخل اليدوي.

## تشخيص شامل للتقارير والملفات التقنية المرجعية

تمت مراجعة وقراءة كل سطر في التقارير المحفوظة في المجلد `/reports` للتأكد من مواءمتها ومطابقتها التامة مع الهيكل الشجري ومعادلات كسر التجربة:

* **ملف `MUBASHER_SERVER_JS_REPORT_AR.md` (تقرير عصب السيرفر):**
  يوضح دقة دمج نصوص الويب Next.js المستقل مع Express، وكيفية تمرير البث عبر الأنبوب البرمجي (Piping) لحماية مصادر البث وتشفير الهوية الفيزيائية للمزودين الأصليين.
* **ملف `MUBASHER_TRIAL_USED_REPORT_AR.md` (تقرير رصد التجربة):**
  يوثق المعادلة الحسابية لتتبع عداد السبعة أيام التنازلي بناءً على الختم الزمني المشفر ومقارنته بمعرف عتاد السيرفر.
* **ملف `MUBASHER_SYS_TIME_REPORT_AR.md` (تقرير ساعة النظام):**
  يشرح الخوارزمية الرياضية للكشف عن تلاعب تواريخ نظام التشغيل وحمايتها.
* **ملفي `MUBASHER_LICENSE_JWT_REPORT_AR.md` و `MUBASHER_LICENSE_PUB_REPORT_AR.md`:**
  يكشفان استخدام تقنية التوقيع الإلكتروني المشفر بمفاتيح التشفير اللا تماثلية لتفادي صياغة تراخيص مزيفة.
