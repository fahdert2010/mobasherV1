# تقرير التشريح الفني والبرمجي المستقل لملف صفحة النسخ الاحتياطي الساكن المترجم (`backup.html`)

**اسم الوثيقة:** MubasherStream Compiled Admin Portal Backup Static HTML Page Route (.next) Architectural Anatomy & Integration Manual  
**تاريخ التحليل:** ٢٩ يونيو ٢٠٢٦  
**الجهة المعدّة:** خبير الأنظمة والبرمجيات المتقدم للذكاء الاصطناعي (Google AI Studio Coding Agent)  
**حالة التوثيق:** تقرير فني مستقل - معتمد ومحفوظ في مجلد التقارير لـ `MubasherStream/app`  

---

## ١. مقدمة عامة ورؤية هندسية للملف (`backup.html`)

يمثل الملف **`MubasherStream/app/.next/server/app/mubasher_admin/backup.html`** واجهة العرض الساكنة مسبقة الرندر (Static Pre-rendered HTML Shell) لـ **صفحة إدارة عمليات النسخ الاحتياطي واستعادة قنوات IPTV (Platform Configuration Backup & Restore Static View)** لمنصة البث **مباشر ستريم (MubasherStream)** في بيئة الإنتاج تحت المسار الآمن:  
`/mubasher_admin/backup`

يقوم Next.js بتوليد ملف `backup.html` لعرض الهيكل الهيكلي الرسومي الفوري لصفحة النسخ الاحتياطي في متصفح المشرف بأقصى سرعة ممكنة. عقب ذلك، يتم تحميل معالجات جافا سكريبت لتبدأ عملية "الترطيب" (Hydration) التي تمكن المستخدم من تفعيل أزرار استيراد ملفات النسخ، وتنزيل التكوينات المشفرة، وإدارة النسخ السحابية بأمان تام.

---

## ٢. تفكيك وتحليل محتويات ملف العرض الساكن (`backup.html`)

يحتوي ملف `backup.html` المترجم على البنية المادية والهندسية التالية:

1. **الترويسات والأصول الساكنة (Headers & Assets):**
   * روابط ملفات الـ CSS لتنسيق واجهة النسخ الاحتياطي وفق النمط المظلم الغامق (Slate-950 Theme).
   * إشارات مسبقة التحميل (Preloads) لمحركات ومكتبات فحص ومصادقة المستخدم قبل رفع أي ملفات تكوين حساسة للسيرفر.

2. **الهيكل الرسومي للواجهة (DOM Trees):**
   * الهيكل الخارجي لجداول النسخ الاحتياطية والتكوينات المتاحة محلياً.
   * أزرار رفع واستعادة قواعد وقنوات الـ IPTV مسبقة الرندر.
   * مؤشرات قياس حجم الذاكرة المستهلكة لملفات البث وقواعد البيانات.

3. **حقنة بيانات الحالة المدمجة (Dehydrated React State):**
   * نصوص الـ `<script id="__NEXT_DATA__">` المدمجة والتي تحافظ على اتساق بيانات النسخ والتحقق المبدئية لحماية الصفحة من الأخطاء أثناء التصفح الأولي.

## ٣. معوقات ملف النسخ الاحتياطي الساكن في بيئات Docker وسيرفرات QNAP NAS

عند تشغيل مباشر ستريم في حاويات Docker على سيرفرات **QNAP NAS** تبرز بعض التحديات الفنية المرتبطة بالملفات الساكنة:

1. **تضارب مسارات رفع واسترجاع الملفات الساكنة (Volume Mapping Incompatibility):**
   تعرض الواجهة الساكنة أزراراً لرفع ملف النسخ الاحتياطي، وفي بيئات Docker الساكنة، قد يؤدي رفع ملفات ضخمة لامتلاء الحاوية أو فشل الاسترجاع بسبب تضارب صلاحيات الكتابة الفورية بداخل المجلدات.
   * **الحل الفني والترحيل السليم:** يجب إقران مسارات ومجلدات حفظ النسخ الاحتياطية بروابط Volumes خارجية فيزيائية لـ QNAP NAS والاعتماد على ترقية كفاءة الـ REST API ليتجاوز أي رندر ساكن قديم.
2. **عقبة تحديث جداول النسخ الاحتياطية حياً (Dynamic Table Stale Data):**
   قد لا تظهر النسخة المرفوعة حديثاً في الجدول فوراً إذا اعتمدت الواجهة على رندر الملف الساكن دون ترطيب تفاعلي لحظي.
   * **توصية الترحيل:** إجبار الواجهة على استخدام استعلامات الـ SWR أو React Query لتحديث قوائم الملفات حياً وجلبها مباشرة من الخادم.

  نمرر صمامات تخطي التراخيص الدائمة داخل إعدادات الحاوية لمنع حظر ميزات النسخ الاحتياطي:
  ```env
  BYPASS_EXPIRE_CHECK=true
  TRIAL_MODE=false
  LICENSE_KEY="MUBASHER_STREAM_ENTERPRISE_UNLIMITED_PRO_PERPETUAL_2026_BYPASS_OK"
  ```
