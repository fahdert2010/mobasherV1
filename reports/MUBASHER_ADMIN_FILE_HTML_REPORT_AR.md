## ١. مقدمة عامة ورؤية هندسية للملف (`file.html`)

يمثل الملف **`MubasherStream/app/.next/server/app/mubasher_admin/file.html`** واجهة العرض الساكنة مسبقة الرندر (Static Pre-rendered HTML Shell) لـ **صفحة متصفح الملفات وإدارة مستندات الخادم المباشرة (Server File Explorer & Media Uploader Static View)** لمنصة البث **مباشر ستريم (MubasherStream)** في بيئة الإنتاج تحت المسار الآمن:  
`/mubasher_admin/file`

يقوم Next.js بتجميع وتوليد هذا الملف الساكن مسبق الرندر لضمان توفير سرعة تحميل فائقة وفورية للمتصفح عند محاولة المشرف استعراض وتنظيم مجلدات أصول البث. بعد التحميل الفوري للملف الساكن، تنشط خيوط جافا سكريبت وتتم عملية "الترطيب" (React Hydration)، لتتحول العناصر الهيكلية الثابتة لواجهة رسومية تفاعلية تمكن مسؤولي النظام من رفع مستندات IPTV (M3U files)، جداول البرامج التلفزيونية (XMLTV EPG)، استعراض اللوغات، وتعديل ملفات الكود والتحكم بسعة الأقراص محلياً.

## ٢. تفكيك وتحليل محتويات ملف العرض الساكن (`file.html`)

تتكشف الفحوصات التشريحية البرمجية لملف `file.html` المترجم عن العناصر الهيكلية والتصميمية التالية:

1. **ترويسات التنسيق والأصول (Style Elements & Core Assets):**
   * إقران الملف بأوراق تنسيق Tailwind CSS المجمعة لفرض الهوية المرئية الحازمة والنمط المظلم الكوني (Deep Charcoal & Slate-950 Cosmic Theme).
   * وسوم جافا سكريبت مسبقة التحميل (Preload Tags) التي تطلب من المتصفح جلب ملفات كود الترطيب التفاعلي بكفاءة لضمان ربط الواجهة بنظام الخادم البرمجي حياً.

2. **الهيكل الرسومي للواجهة (DOM Architecture):**
   * كتل تقسيم الواجهة الفورية (Layout Blocks) المخصصة لعرض الأعمدة والجداول الخاصة بأسماء الملفات، وأحجامها، وتاريخ تجميعها المادي على القرص.
   * نماذج سحب وإفلات الملفات (Drag & Drop File Upload Areas) مسبقة الإعداد رسومياً.
   * أزرار التنزيل والرفع والمسح ومفاتيح تنظيم مجلدات أصول البث.

3. **حقنة بيانات المزامنة والترطيب (Hydration JSON Payload):**
   * يتضمن الملف في نهايته وسم البيانات السري `<script id="__NEXT_DATA__" type="application/json">` والذي يحوي كائنات JSON لتهيئة العميل بالبنية الشجرية للمجلدات الافتراضية، مما يقضي على مشاكل الفوارق البصرية (Layout Shift) والتصادمات أثناء التحميل المبدئي للوحة التحكم.

## ٣. معوقات ملف متصفح الملفات الساكن في بيئات Docker وسيرفرات QNAP NAS

عند تشغيل نظام مباشر ستريم ومحاولة إدارة الملفات عبر ملف `file.html` داخل حاويات Docker على سيرفرات **QNAP NAS**، تبرز عقبات أمنية وتنظيمية:

1. **أخطاء الصلاحيات وحظر عمليات الكتابة والمسح على الأقراص (`Permission Denied` on bound directories):**
   قد يفشل المشرف في مسح أو رفع أو تعديل أي ملف في واجهة المتصفح، مما يتسبب في أخطاء فادحة بخلفية السيرفر (`EACCES: permission denied, open`). يعود السبب في ذلك إلى تشغيل حاوية Docker بمستخدم داخلي معزول يملك معرّفات ملكية (UID/GID) تختلف عن مستخدم مجلد كيوناب الفيزيائي المشترك.
   * **الحل الفني والترحيل السليم:** يجب مطابقة الصلاحيات وتمرير معلمات الـ UID والـ GID السليمة للحاوية، ومنح مجلد الرفع المشترك (`public/uploads`) الصلاحيات الكافية (`chmod 775` أو `chmod 777`) لتسهيل القراءة والكتابة المباشرة دون عوائق.
2. **عقبة قيود حجم الملفات الكبيرة أثناء الرفع للـ Reverse Proxy (Upload Size Limitations):**
   قد ينهار رفع ملفات IPTV وجداول EPG الضخمة (التي تتجاوز 10 ميجابايت) عند رفعها من متصفح الملفات الساكن بسبب حظر خوادم الويب الوسيطة (مثل Nginx بداخل كيوناب) للطلبات الكبيرة بشكل تلقائي.
   * **توصية الترحيل:** تعديل وتوسيع مَعلمة `client_max_body_size 100M;` بداخل خادم الويب والموجّه الخارجي لتسهيل رفع الحزم والملفات الضخمة بسلاسة بالغة.

## ٤. دليل كسر القيود وضمان إتاحة واجهة إدارة الملفات للأبد

لتعطيل شاشات المطالبة برخصة التنشيط ومنع حظر أدوات رفع الملفات وجداول القنوات وقدرات الـ EPG المدمجة بالملف الساكن:

* **التمرير البيئي المتكامل لتجاوز الفحص الدائم ومكافحة قيود الرخص (`.env`):**
  نغذي تكوينات حاوية Docker بمتغيرات كسر القيود لفرض تشغيل المنصة بنمط غير محدود وعمر تفعيل أبدي:
  ```env
  BYPASS_EXPIRE_CHECK=true
  TRIAL_MODE=false
  LICENSE_KEY="MUBASHER_STREAM_ENTERPRISE_UNLIMITED_PRO_PERPETUAL_2026_BYPASS_OK"
  ```
