## ١. مقدمة عامة ودور سكربت توليد الأسرار الأمنية (Introduction & Role of the Script)
لحماية قنوات البث الحية وقواعد البيانات وحماية لوحات التحكم من الاختراق، يتوجب على المشغلين استخدام سلاسل أمنية مشفرة شديدة التعقيد لحماية ملف الإعدادات البيئية من هجمات التخمين ومحاولات انتحال الشخصية.

يمثل السكربت الخدمي المساعد **`generate-secret.js`** أداة سريعة في بيئة تشغيل نود جي إس مصممة لمساعدة مشرف النظام أو المطور عند إعداد السيرفر لأول مرة لإنتاج مفاتيح تشفير عشوائية قوية مبرهنة رياضياً للأمن السيبراني، ومن ثم تضمينها داخل ملفات الكوكي وجلسات التحقق الـ JWT.

بخلاف استخدام التخمين البشري الذي غالباً ما يؤدي لإنتاج كلمات مرور ضعيفة، يضمن هذا السكربت توليد سلاسل تشفير مشفرة غاية في القوة ومستحيلة الفك عبر الاعتماد على خوارزميات التوليد العشوائي المتكاملة في عتاد المعالج.

## ٢. الهيكل البرمجي والبروتوكول الداخلي التام للسكربت (`generate-secret.js`)

يمتاز كود السكربت بالبساطة البرمجية العالية والاعتماد الكلي على حزم نود الافتراضية لضمان سرعة العمل بدون الحاجة لتثبيت أي حزم تابعة خارجية معقدة:

```javascript
/**
 * ====================================================================
 * MUBAHSER STREAM ENTERPRISE - CRYPTOGRAPHIC SECRET KEY GENERATOR
 * ====================================================================
 * وظيفته: توليد سلاسل مشفرة عشوائية عالية القوة لتأمين الكوكيز والترخيص في ملف .env
 */

const crypto = require('crypto');
const fs = require('fs');
const path = require('path');

// ١. تحديد القوة التشفيرية المطلوبة بالبايت
const SECRET_SIZE_BYTES = 32; // ينتج مفاتيح بطول ٢٥٦ بت (64 حرف هكس)
const COOKIE_SIZE_BYTES = 16; // ينتج مفاتيح بطول ١٢٨ بت (32 حرف هكس)

function generateSecureSecret(size) {
    // استخدام محرك الأمان التابع لنواة المعالج لإنتاج عشوائية مشفرة حقيقية
    return crypto.randomBytes(size).toString('hex');
}

function initSecrets() {
    console.log("\n🔑 [MubasherStream] Starting Cryptographic Key Generator...");
    
    const jwtSecret = generateSecureSecret(SECRET_SIZE_BYTES);
    const cookieSecret = generateSecureSecret(COOKIE_SIZE_BYTES);
    
    console.log("----------------------------------------------------------------");
    console.log(`✨ Generated JWT_SECRET_KEY:       ${jwtSecret}`);
    console.log(`✨ Generated COOKIE_ENCRYPTION_KEY: ${cookieSecret}`);
    console.log("----------------------------------------------------------------");
    
    const envPath = path.join(__dirname, '.env');
    
    // ٢. آلية الكتابة التلقائية لملف الإعدادات البيئية لسهولة الإعداد
    if (fs.existsSync(envPath)) {
        let envContent = fs.readFileSync(envPath, 'utf8');
        
        // استبدال قيم التشفير الافتراضية بالقيم الحقيقية الجديدة القوية
        envContent = envContent.replace(
            /JWT_SECRET_KEY=.*/, 
            `JWT_SECRET_KEY="${jwtSecret}"`
        );
        envContent = envContent.replace(
            /COOKIE_ENCRYPTION_KEY=.*/, 
            `COOKIE_ENCRYPTION_KEY="${cookieSecret}"`
        );
        
        fs.writeFileSync(envPath, envContent, 'utf8');
        console.log("💾 [Success] Automatically updated and hardened your root '.env' file keys.");
    } else {
        console.log("⚠️ [Warning] '.env' file not found at the root level. Please copy-paste keys manually.");
    }
    console.log("🚀 System Security Hardened Successfully!\n");
}

initSecrets();
```

## ٣. تشريح المكونات والأكواد الرياضية في السكربت (Deep Code Technical Anatomy)

نفكك السطور ونبين الوظيفة التقنية الدقيقة لعمليات التشفير:

### أ. استدعاء مكتبة الأمان السيبراني المدمجة (`crypto`)
* يرفض الكود استخدام دوال التوليد العشوائي العادية مثل `Math.random()` لأنها تعتمد على معادلات حسابية متكررة غير آمنة يسهل تخمينها برمجياً (Pseudo-random number generators).
* يستعين عوضاً عن ذلك بـ **`require('crypto')`** التي تتواصل مباشرة مع منشئ الأرقام العشوائية المشفرة لنظام التشغيل (Cryptographically Secure Pseudo-Random Number Generator - CSPRNG)، مما يضمن عدم تكرار المفاتيح الناتجة على مستوى الكون لمليارات السنين.

### ب. دالة توليد الأرقام العشوائية المشفرة (`crypto.randomBytes`)
* **`crypto.randomBytes(size)`**: تولد بايتات ثنائية عشوائية حقيقية من الذاكرة الرام العشوائية ومن حركة المعالج.
* **`toString('hex')`**: تحول البايتات الثنائية المعقدة إلى سلسلة أحرف ستة عشرية (Hexadecimal String) قابلة للمشاهدة والكتابة داخل ملفات نصوص التكوين `.env` دون حدوث تشوه للبيانات.

### ج. محرك المسح التلقائي والاستبدال الذكي (Regex Auto-Updater)
* يسهل السكربت على مشرف خوادم كيوناب QNAP عملية التهيئة، فبدل الاكتفاء بعرض الرموز في الشاشة البرمجية ومطالبته بنسخها ولصقها، يحدد السكربت مسار ملف `.env` عبر دالة المسار `path.join(__dirname, '.env')`.
* يستعمل تعابير المطابقة المنتظمة (Regular Expressions) مثل `/JWT_SECRET_KEY=.*/` للبحث الديناميكي السريع عن السطر المخصص للمفتاح واستبداله بقيمته المولدة الجديدة الفائقة الأمان وحفظ الملف دفعة واحدة بأمان تام.

## ٥. تفاصيل الأمان والصيانة في السكربت (Security Best Practices)

* **القوة التشغيلية:** يوصى بتشغيل هذا السكربت لمرة واحدة عند التهيئة الأولى للنظام، ويستحسن إعادة تشغيله كل ٦ أشهر من باب الصيانة الوقائية (Key Rotation) لضمان قتل وتدمير أي جلسات قديمة قد تكون بقيت معلقة في أجهزة المشغلين السابقين أو المشتبه بهم.
* **الأثر الإيجابي على QNAP:** بتشغيل السكربت داخل حاوية Docker، نضمن توليد أسرار محلية فريدة وخاصة بالعميل تماماً ومستحيلة التكرار في أي نسخة تشغيلية أخرى لمباشر ستريم على خوادم كيوناب الأخرى حول العالم.
