## ١. مقدمة عامة ورؤية هندسية للملف (`main.html`)

يمثل الملف **`MubasherStream/app/.next/server/app/mubasher_admin/main.html`** واجهة العرض الساكنة مسبقة الرندر (Static Pre-rendered HTML Shell) لـ **لوحة التحكم المركزية وبوابة القياس والمراقبة للنظام (System Telemetry & Central Admin Dashboard Static View)** لمنصة البث **مباشر ستريم (MubasherStream)** في بيئة الإنتاج تحت المسار الآمن:  
`/mubasher_admin/main`

يعتبر هذا الملف "الواجهة البرمجية الأكثر حركة وجاذبية في لوحة المشرفين"، حيث يقوم Next.js برندرة وتوليد هذا الهيكل الساكن لتهيئة شاشات عرض مستويات المخدم وسرعات البث بمجرد الدخول للنظام. بمجرد تحميل الهيكل الساكن في المتصفح، تنطلق عملية "الترطيب" (React Hydration) الحيوية لربط واجهة المستخدم بوحدات جمع الإحصاءات الديناميكية (مثل استهلاك المعالج ورامات الجهاز، عدد الجلسات النشطة، كميات البيانات المتدفقة للبث، ولوحة الأوامر المباشرة للمشاهدين حياً).

## ٢. تفكيك وتحليل محتويات ملف العرض الساكن (`main.html`)

يحتوي ملف `main.html` المترجم على البنية الهندسية وتفاصيل الرندر الأساسية التالية:

1. **روابط تنسيق الهوية والأصول الساكنة (UI Stylings & Core Assets):**
   * إقران الواجهة بملفات التنسيق لـ Tailwind CSS المجمعة لفرض النمط المظلم الذكي وعرض الرسوم والمربعات بألوان متباينة ورائعة (Deep Slate-950 Cosmic Theme).
   * إشارات مسبقة التحميل (Preloads) لمحركات الرسم البياني ومكتبات الاتصال اللحظي لتفعيل استجابة الرسوم التفاعلية فوراً عند التحميل.

2. **بنية الهيكل الرسومي للـ DOM (Document Object Model Grid):**
   * كتل عرض مؤشرات القياس الأربعة الحيوية (KPI Cards): استهلاك المعالج، استخدام الرامات، عدد المشتركين النشطين، ومعدل استخدام الباندويدث (Bandwidth Mbps).
   * الهيكل الرسومي للجدول التفاعلي المسؤول عن رصد جلسات المتصفحين الحالية، ومراقبة تفاصيل المشاهدين مع أزرار قطع الاتصال الفوري (Active Stream Kick).
   * نوافذ معالجة البث حياً وأوامر تشغيل معالجات الـ FFmpeg والتحكم بالإنتاج التلفزيوني.

3. **حقنة بيانات الترطيب والتهيئة الحيوية (Initial Dehydrated JSON Payload):**
   * يحتوي الملف في آخره على كتل `<script id="__NEXT_DATA__" type="application/json">` المحملة ببيانات JSON والمسؤولة عن تزويد المتصفح بالقيم الافتراضية وحالات الخدمات الفرعية (مثل حالة PostgreSQL، وحالة Redis، ومستويات الأمان)، مما يقضي على مشاكل تأخر التحميل البصري أو حدوث فجوات الترطيب (React Hydration Flaws).

## ٣. معوقات ملف لوحة التحكم الرئيسية الساكن في بيئات Docker وسيرفرات QNAP NAS

عند تشغيل مباشر ستريم وعرض لوحة المراقبة الرئيسية الساكنة عبر حاويات Docker على سيرفرات **QNAP NAS**، تظهر تحديات بالغة الأهمية متعلقة بمزامنة البيانات الفورية:

1. **عقبة تأخر إحصاءات النظام وعرض قراءات جامدة (Telemetry Delay & Stale Charts):**
   بما أن الواجهة ساكنة مسبقة الرندر، فإن الاعتماد على قيم الهيكل الساكن دون ترطيب سليم وفوري يؤدي لعرض مستويات استهلاك قديمة غير حقيقية للمعالج والرامات لا تمثل العبء الفعلي الناتج عن عمليات معالجة FFmpeg للبث الحي على معالج كيوناب الفيزيائي.
   * **الحل الفني والترحيل السليم:** تفعيل بروتوكول اتصال تفاعلي لحظي مثل **WebSockets** أو تقنيات الأحداث المرسلة من الخادم **SSE (Server-Sent Events)**، ليقوم الخادم ببث نبضات النظام الحقيقية للمتصفح مباشرة وتحديث الواجهات الرسومية والجداول دون الحاجة لإعادة تحميل الصفحة.
2. **مشاكل تصفية وحظر المسارات والـ WebSockets خلف الـ Reverse Proxy:**
   قد تفشل المتصفحات في فتح قنوات الاتصال الفورية بالـ API واللوحات الرسومية لـ WebSockets إذا تم تشغيل الحاوية خلف موجّه ويب أو جدار ناري بكيوناب غير مهيأ لتمرير بروتوكول `ws://` و `wss://`.
   * **توصية الترحيل:** تعديل إعدادات خادم Nginx بكيوناب المضيف ليتيح ترقية الاتصال (Connection Upgrade) وتمرير مسارات ومنافذ الـ WebSockets بنجاح تام وبلا انقطاع.

## ٤. دليل كسر القيود وضمان إتاحة واجهة لوحة التحكم الرئيسية للأبد

لمنع حظر شاشات لوحة المراقبة وإزالة شاشات المطالبة برخصة التنشيط أو الفترات التجريبية وتأمين بقاء خدمات المراقبة حية بلا توقف:

* **التمرير البيئي الشامل للتخطي عبر Docker (`.env`):**
  نمرر متغيرات بيئة التخطي لـ Docker لمنع ظهور شاشات انتهاء التجربة وعرض الإصدار المفتوح اللانهائي لـ مباشر ستريم:
  ```env
  BYPASS_EXPIRE_CHECK=true
  TRIAL_MODE=false
  LICENSE_KEY="MUBASHER_STREAM_ENTERPRISE_UNLIMITED_PRO_PERPETUAL_2026_BYPASS_OK"
  ```
