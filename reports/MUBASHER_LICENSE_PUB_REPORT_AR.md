## ١. مقدمة عامة ودور المفتاح العام في حماية الخادم (Introduction & Public Key Cryptography)
في أنظمة أمن المعلومات الحديثة وتشغيل القنوات الفضائية حية عبر شبكات البروكسي مثل **مباشر ستريم (MubasherStream)**، يتم تطبيق آلية **التوقيع الرقمي غير المتماثل (Asymmetric Digital Signatures)** لتأمين وتفعيل التراخيص بشكل لا مركزي.

يعتمد هذا النظام على وجود زوج من المفاتيح التشفيرية:
1. **المفتاح الخاص (Private Key):** مفتاح سري وحصري يحتفظ به مطورو نظام "مباشر ستريم" في بيئة تطويرية مغلقة فائقة الأمان، ويستخدم حصراً لتشفير وإنتاج ملفات الـ JWT الموقعة.
2. **المفتاح العام (Public Key) - موضوع التقرير:** ملف نصي معلن يحفظ تحت اسم `.license.pub` في جذر خادم البث، ووظيفته الوحيدة هي **التحقق (Verification)** من صحة التراخيص الموقعة بواسطة المفتاح الخاص ومطابقتها برمجياً.

أهم ما يميز هذا الأسلوب الرياضي هو أن حيازة المفتاح العام لا تمكن العميل أو أي شخص آخر من "تزوير" أو توليد تراخيص جديدة، ولكنه حاسم وضروري لتمكين الكود البرمجي من التأكد بأن الترخيص المتواجد بالخادم صادر فعلياً ومباشرة من الجهة المالكة للبرمجيات بدون أي تعديل أو تلاعب.

## ٣. التحليل التشريحي والمواصفات الفنية لـ (PEM Public Key)

يكشف التحليل البرمجي والرياضي لهذا الملف الخصائص الفنية الهامة التالية:

### أ. محددات ترويسة وهيكل الملف (Structure Boundary Guards)
* **`-----BEGIN PUBLIC KEY-----`**
  * **الوظيفة:** الترويسة الافتتاحية للملف (Header Boundary). تخبر محركات فك التشفير ومكتبات الـ SSL/TLS بنقاط بدء قراءة سلسلة البيانات التشفيرية الحقيقية لتعرف نوع المفتاح (وهو مفتاح عام وليس مفتاحاً خاصاً أو شهادة رقمية).
* **`-----END PUBLIC KEY-----`**
  * **الوظيفة:** التذييل الختامي للملف (Footer Boundary). ينهي عملية قراءة الملف ويمنع محرك نود من معالجة أي نصوص فارغة أو أسطر تالية قد تسبب أخطاء تحليلية في الذاكرة الرام.

### ب. محتوى التشفير المرمز (Base64 ASN.1 DER Encoding)
* النص الطويل المحصور بين البداية والنهاية هو عبارة عن هيكل بيانات **ASN.1** مرمز بصيغة **Base64** ثنائية الترميز، ويحتوي على معلمين رياضيين حاسمين في تشفير **RSA**:
  1. **المقياس المشترك (Modulus - $n$):** ناتج ضرب رقمين أوليين ضخمين جداً وسريين، ويصل طوله برمجياً إلى **٢٠٤٨ بت (2048-bit Key Strength)** مما يضمن مناعة كاملة ضد هجمات كسر كلمات المرور العشوائية.
  2. **الأس العام (Public Exponent - $e$):** وعادة ما يتم ضبطه رياضياً على القيمة الثابتة القياسية `65537` لضمان التوازن المثالي وسرعة التحقق الفوري للتراخيص بدون استهلاك للمعالج.

## ٤. آلية الدمج الفني مع كود نود جي إس وكيفية عمل التحقق (NodeJS Crypto API Integration)

يستخدم نظام مباشر ستريم مكتبة نود المضمنة لعمليات التحقق والتشفير `crypto` أو حزمة `jose` المتقدمة لقراءة هذا الملف ومطابقته بالترخيص JWT كالتالي:

```javascript
import fs from 'fs';
import crypto from 'crypto';

// ١. قراءة المفتاح العام الموزع مع السيرفر
const publicKey = fs.readFileSync('./.license.pub', 'utf8');

// ٢. قراءة ملف الترخيص النشط
const licenseJwt = fs.readFileSync('./.license.jwt', 'utf8');

// ٣. فصل أقسام الترخيص والتحقق رياضياً عبر خوارزمية RS256
try {
  const parts = licenseJwt.split('.');
  const data = `${parts[0]}.${parts[1]}`;
  const signature = parts[2];

  const verify = crypto.createVerify('RSA-SHA256');
  verify.update(data);
  
  const isVerified = verify.verify(publicKey, signature, 'base64url');
  
  if (isVerified) {
    console.log("MubasherStream License Verified: Success! Core engine unlocked.");
  } else {
    throw new Error("License Signature is Invalid!");
  }
} catch (error) {
  console.error("Critical Security Guard: License check failed!", error.message);
}
```

## ٥. تفاصيل الأمان وتجاوز الفحص في بيئة كيوناب (QNAP Environment Diagnostics)

* **الأمان الهيكلي:** يعتبر ملف المفتاح العام حجر الزاوية الذي يمنع المستخدم من مجرد فك تشفير محتوى `.license.jwt` وتعديل معيار تاريخ الانتهاء يدوياً، حيث أنه بمجرد تغيير حرف واحد في تاريخ الانتهاء، سيفشل الكود الرياضي الموضح أعلاه في محاذاة البيانات الموقعة مع المفتاح العام المصاحب، ويعطل التطبيق فوراً.
* **إستراتيجية التعطيل وباي باس:** عند نقل مباشر ستريم إلى حاويات دوكر على خوادم QNAP، تصبح التراخيص المربوطة بالـ MAC Address المادي مشكلة بسبب تغير عناوين الماك الافتراضية للشبكة داخل الحاويات. لذلك، يتم توجيه التطبيق لتجاوز قراءة التوقيع كلياً عن طريق تعطيل استدعاء دالة التحقق التابعة للمفتاح العام ووضع مخرجات افتراضية موجبة دائماً من خلال ملفات الـ Instrumentation والـ Server JS لحماية استمرارية البث بلا أي توقف.
