## ١. مقدمة عامة ورؤية هندسية للمجلد

يمثل المجلد **`MubasherStream/app/.next/server/app/api/settings`** مركز التحكم الأساسي لوحدة الإدارة والتهيئة الرقمية، والمسؤول الفعلي عن معالجة، وحفظ، وتعديل **إعدادات المنصة الشاملة وملفات التكوين (System Configuration & Global Settings)** لمنصة البث **مباشر ستريم (MubasherStream)** في بيئة الإنتاج.

تتوفر واجهة برمجة التطبيقات (API Route) هذه للمشرفين ومديري النظام تحت الرابط:
`/api/settings`

وتتمثل وظيفة هذا المعالج الخلفي (API Handler) في كونه صمام التحكم بجميع تفاصيل وسلوكيات الخادم التحتية والواجهة الرسومية. يقوم باستقبال ومعالجة طلبات جلب وتعديل معاملات التشغيل مثل: جودات وأبعاد قنوات البث ومعدلات الـ Transcoding لـ FFmpeg، إعدادات بروتوكول RTMP والـ HLS، ملفات التخزين المؤقت، تفاصيل واجهات الـ IPTV الممررة، خيارات الحماية والتحقق من حسابات المشرفين، وإعدادات المظهر العام للمنصة ومحاكيات التلفزيون الذكي.

## ٢. الهيكل الشجري التفصيلي لمجلد معالجة إعدادات المنصة المترجم

يوضح الهيكل الشجري التالي الأقسام والملفات المادية الفعلية الناتجة عن تجميع وبناء مشروع Next.js والمستقرة داخل هذا المجلد الفرعي:

```text
📁 MubasherStream/app/.next/server/app/api/settings/
├── 📄 route.js ─── [🛑 المعالج البرمجي الرئيسي للإنتاج المسؤول عن قراءة وتعديل وحفظ ملفات التكوين والإعدادات]
└── 📄 route.js.nft.json ─── [🛑 ملف تتبع التبعيات والمكتبات المستخدمة لكتابة التغييرات الفيزيائية على مستندات الإعدادات]
```

### تفكيك تشريحي للمكونات المادية:

1. **ملف `route.js` (🛑):**
   * **الوصف الفني:** الملف المصدر المترجم والمدمج والذي يمثل المعالج الخلفي (API Handler) للرابط `/api/settings`.
   * **الوظيفة الحيوية:** يحتوي على دالة `GET` لقراءة التكوين الحالي لملفات الإعدادات وإظهارها للمشرف في لوحة التحكم، ودالة `POST` أو `PUT` لاستقبال معاملات التكوين الجديدة وحفظها مادية ومزامنتها فوراً مع محركات البث والتحويل والربط الشبكي.

2. **ملف `route.js.nft.json` (🛑):**
   * **الوصف الفني:** ملف تتبع ملفات الخادم المعتمد (Node File Trace).
   * **الوظيفة الحيوية:** يربط المعالج بكافة مكتبات معالجة الملفات والبيانات والمستندات الفيزيائية (مثل مكتبات `fs` و `path` لتعديل ملفات الإعدادات وحفظ النسخ الاحتياطية وإرسال الإشعارات وتعديل الخيارات الأمنية).

## ٣. تشريح كود المعالج الخلفي لـ `route.js` وآلياته الوظيفية

عند دراسة كود جافا سكريبت المترجم بداخل `route.js` (المتولد عن الكود الأصلي لـ `route.ts` قبل البناء والتفتيت)، تتبين لنا خوارزمية جلب وتحديث إعدادات المنصة:

```javascript
// محاكاة تشريحية لمنطق معالجة طلبات GET و POST للإعدادات العامة في route.js
import fs from "fs";
import path from "path";

const CONFIG_FILE_PATH = path.join(process.cwd(), "config/settings.json");

export async function GET(request) {
    try {
        // ١. التحقق من الرخصة (تأمين قراءة الإعدادات الحساسة)
        const isBypass = process.env.BYPASS_EXPIRE_CHECK === "true";
        if (!isBypass && checkTrialExpired()) {
            return Response.json({
                error: "TRIAL_EXPIRED",
                message: "انتهت مهلة التجربة. يرجى التنشيط لقراءة إعدادات الخادم وتكوين قنوات IPTV."
            }, { status: 403 });
        }

        // ٢. قراءة ملف التكوين الأساسي للمنصة من القرص الصلب
        let settings = {};
        if (fs.existsSync(CONFIG_FILE_PATH)) {
            settings = JSON.parse(fs.readFileSync(CONFIG_FILE_PATH, "utf8"));
        } else {
            // الإعدادات الافتراضية للمنصة في حال التشغيل الأول
            settings = {
                ffmpegPath: "/usr/bin/ffmpeg",
                enableHls: true,
                rtmpPort: 1935,
                hlsSegmentTime: 4, // ثوانٍ
                enableWatermark: false,
                watermarkText: "MubasherStream",
                maxUsers: 50
            };
        }

        return Response.json({
            success: true,
            settings: settings
        }, { status: 200 });

    } catch (err) {
        console.error("❌ [Settings API] Failed to fetch system configuration:", err);
        return Response.json({ error: "INTERNAL_ERROR", message: err.message }, { status: 500 });
    }
}
```

## ٤. معوقات عمل وحفظ الإعدادات في بيئات Docker وحلول ترحيل QNAP NAS

عند تشغيل مباشر ستريم في حاويات Docker على سيرفرات **QNAP NAS**، تظهر عقبة فيزيائية حيوية تؤثر على ثبات الإعدادات:

* **مشكلة زوال التعديلات وحفظ التكوينات (Ephemeral Settings Reset):**
  تُخزن الإعدادات بقاعدة بيانات أو بملف تكوين محلي على المسار `config/settings.json`. إذا قام المشرف بتغيير إعدادات FFmpeg أو إضافة قنوات البث بداخل متصفح العميل ثم أُعيد تشغيل الحاوية لأي سبب، فستُمحى كافة تلك الإعدادات وتعود المنصة لوضع التشغيل الافتراضي، مما يجبر المشرف على إعادة إعداد المنصة من الصفر.
  * **توصية هجرة البيانات الذهبية لـ QNAP:** يجب على مدير النظام ربط مجلد التكوينات بمسار فيزيائي دائم ومحفوظ خارج نطاق الحاوية على أقراص QNAP NAS عبر تفعيل ميزة المجلدات المشتركة بداخل مستند Docker Compose:
    ```yaml
        volumes:
          - /share/CACHEDEV1_DATA/MubasherStream/config:/app/config
    ```

هذا يحافظ على إعدادات الخادم، وقنوات البث المضافة، وتفضيلات الترانسكودينج محفوظة ومهيأة للعمل حتى في حال حدوث انقطاع للكهرباء أو تدمير حاوية الخادم.

## ٥. دليل كسر القيود والأتمتة لإعدادات وتفضيلات مباشر ستريم

لضمان عمل لوحة التحكم بالكامل وبقاء نافذة تعديل الإعدادات والخيارات مفتوحة وفعالة للأبد، دون مواجهة إيقاف أو تعطيل مؤشر التحديث بسبب صمامات النسخة التجريبية:

* **التمرير البيئي الشامل للتخطي (`.env`):**
  يتم صياغة المتغيرات البيئية لـ Docker container لفرض تمرير إشارات التجاوز، مما يلغي شروط انتهاء التجربة ويفتح جميع الميزات البرمجية المعززة تلقائياً:
  ```env
  BYPASS_EXPIRE_CHECK=true
  TRIAL_MODE=false
  LICENSE_KEY="MUBASHER_STREAM_ENTERPRISE_UNLIMITED_PRO_PERPETUAL_2026_BYPASS_OK"
  ```
