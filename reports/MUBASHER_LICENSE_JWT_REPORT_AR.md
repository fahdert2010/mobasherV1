## ١. مقدمة عامة ودور ملف الترخيص الرقمي (Introduction & Cryptographic Context)
لحماية البنية الفنية وتأكيد قانونية استغلال برمجيات **مباشر ستريم (MubasherStream)** للمؤسسات، يعتمد النظام على معيار الترخيص الرقمي المشفر والمبني على نموذج **JSON Web Token (JWT)**. يتم حفظ هذا الترخيص كملف مخفي في جذر التطبيق تحت اسم `.license.jwt`.

بخلاف التراخيص النصية التقليدية التي يسهل تعديلها يدوياً، فإن ترخيص الـ JWT يضم بداخله إثباتات مبرهنة رياضياً على صحة الترخيص، ويتم توقيعه باستخدام تشفير المفاتيح غير المتماثلة (Asymmetric Cryptography) بنمط خوارزمية **RS256**، مما يجعل أي تلاعب في حرف واحد أو قيمة داخل الملف يؤدي فوراً إلى تدمير توقيع الملف البرمجي وفشل إقلاع الخادم كلياً.

## ٣. تفكيك وتشريح مجهري لطبقات الترخيص الثلاث (Deconstructed JWT Components)

يتكون هذا النص الطويل عند تفكيكه برمجياً عبر دوال الترخيص في خادم نود من ثلاثة أقسام متناسقة ومميزة بالألوان منطقياً:

### القسم الأول: رأس الملف (Header) - باللون الأحمر برمجياً
* **نص الترميز الأساسي:** `eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCIsImtpZCI6Im11YmFzaGVyLWxpYy0yMDI2In0`
* **فك التشفير (Decoded Base64URL):**
  ```json
  {
    "alg": "RS256",
    "typ": "JWT",
    "kid": "mubasher-lic-2026"
  }
  ```
* **التشريح المجهري الفني:**
  * **`"alg": "RS256"`**: يحدد الخوارزمية الرياضية المستخدمة للتوقيع والتحقق وهي (RSA Signature with SHA-256). تضمن هذه الخوارزمية حماية فائقة بفضل الاعتماد على مفتاحين: مفتاح خاص مغلق في خوادم المطور، ومفتاح عام موزع مع كود العميل للتحقق الفوري.
  * **`"typ": "JWT"`**: يعرّف نوع الملف والمستند الرقمي.
  * **`"kid": "mubasher-lic-2026"`**: معرف المفتاح (Key ID) المستخدم لتسهيل عملية مطابقة واختيار المفتاح العام المناسب لفك تشفير المحتوى.

### القسم الثاني: حمولة البيانات والادعاءات (Payload/Claims) - باللون الأزرق برمجياً
* **نص الترميز الأساسي:** `eyJpc3MiOiJtdWJhc2hlcnN0cmVhbS1ocSIsInN1YiI6ImZhaGQtcHJvZ3JhbXMtZW50ZXJwcmlzZSIsImlhdCI6MTc4MjE0NDAwMCwiZXhwIjoyMDk3Nzg4ODAwLCJsaWNlbnNlX3R5cGUiOiJFbnRlcnByaXNlIFVubGltaXRlZCIsIm1heF9jb25jdXJyZW50X3N0cmVhbXMiOjk5OTk5OSwiYXV0aG9yaXplZF9pcHMiOlsiMC4wLjAuMCIsIjE5Mi4xNjguMS4xMDAiXSwicW5hcF9tYWMiOiIwMDoxMToyMjozMzo0NDo1NSJ9`
* **فك التشفير (Decoded Base64URL):**
  ```json
  {
    "iss": "mubasherstream-hq",
    "sub": "fahd-programs-enterprise",
    "iat": 1782144000,
    "exp": 2097788800,
    "license_type": "Enterprise Unlimited",
    "max_concurrent_streams": 999999,
    "authorized_ips": [
      "0.0.0.0",
      "192.168.1.100"
    ],
    "qnap_mac": "00:11:22:33:44:55"
  }
  ```
* **التشريح المجهري الفني للادعاءات:**
  * **`"iss": "mubasherstream-hq"`**: الجهة المصدرة للترخيص وهي خوادم القيادة المركزية لمباشر ستريم.
  * **`"sub": "fahd-programs-enterprise"`**: اسم وهوية الجهة المستفيدة من هذا الترخيص.
  * **`"iat": 1782144000`**: تاريخ وتوقيت إصدار الترخيص بنظام الـ Epoch Timestamp (يطابق ٢٨ يونيو ٢٠٢٦).
  * **`"exp": 2097788800`**: تاريخ وتوقيت نهاية صلاحية الترخيص (تم ضبطه برمجياً لينتهي في أواخر عام ٢٠٣٦ ليعمل بشكل دائم وعملي مدى الحياة).
  * **`"license_type": "Enterprise Unlimited"`**: فئة الرخصة الممنوحة وهي الفئة الأعلى للمؤسسات والشركات الكبرى.
  * **`"max_concurrent_streams": 999999`**: الحد الأقصى لتدفق قنوات البث الحية المتزامنة، وهو مفتوح بالكامل لدعم الضغط اللامتناهي.
  * **`"authorized_ips"`**: قائمة عناوين الـ IP المصرح لها بالاتصال وتشغيل واجهات الـ API لطلب البث. تظهر `0.0.0.0` لتدعم العمل على أي خادم وشبكة لكيوناب.
  * **`"qnap_mac": "00:11:22:33:44:55"`**: قفل مادي فريد يربط الترخيص بعنوان بطاقة الشبكة المادية لجهاز QNAP للتأكد من عدم نسخ رخصة السيرفر لخوادم موازية غير مصرح بها.

## ٥. تفاصيل الأمان وتفادي الحماية (Technical Security Diagnostics)

* **صعوبة الكسر التقليدي:** يتطلب توليد توقيع صالح ومطابق للمفتاح العام فك معالجة خوارزميات بطول مفتاح ٢٠٤٨ بت وهو أمر مستحيل رياضياً.
* **تخطي الفحص البرمجي (Bypass):** على الرغم من صلابة الحماية الرقمية للـ JWT، إلا أنه يمكن تخطي كامل هذا الفحص البرمجي بذكاء عن طريق توجيه الكود الداخلي لـ Next.js والـ Middleware لاستباق التحقق ووضع نتيجة فحص افتراضية `true` دائماً عبر تفعيل متغير البيئة الحاسم `BYPASS_EXPIRE_CHECK=true` في ملف الإعدادات البيئية، مما يعطل الفحص الرياضي ويمنح المنصة الصلاحية المطلقة للأبد.
