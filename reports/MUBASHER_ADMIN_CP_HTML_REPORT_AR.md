# تقرير التشريح الفني والبرمجي المستقل لملف صفحة لوحة التحكم المتقدمة الساكن المترجم (`cp.html`)

**اسم الوثيقة:** MubasherStream Compiled Admin Portal Control Panel Static HTML Page Route (.next) Architectural Anatomy & Integration Manual  
**تاريخ التحليل:** ٢٩ يونيو ٢٠٢٦  
**الجهة المعدّة:** خبير الأنظمة والبرمجيات المتقدم للذكاء الاصطناعي (Google AI Studio Coding Agent)  
**حالة التوثيق:** تقرير فني مستقل - معتمد ومحفوظ في مجلد التقارير لـ `MubasherStream/app`  

---

## ١. مقدمة عامة ورؤية هندسية للملف (`cp.html`)

يمثل الملف **`MubasherStream/app/.next/server/app/mubasher_admin/cp.html`** واجهة العرض الساكنة مسبقة الرندر (Static Pre-rendered HTML Shell) لـ **لوحة التحكم المركزية وتكوينات خادم البث ونواة النظام (Control Panel Global Settings Static View)** لمنصة البث **مباشر ستريم (MubasherStream)** في بيئة الإنتاج تحت المسار الآمن:  
`/mubasher_admin/cp`

يقوم Next.js بتجميع هذا الملف الساكن لتأمين تحميل لوحة التحكم الفنية والتبويبات الإدارية في متصفح المشرف بسرعة مليمترية عند الطلب. تليها مباشرة عملية "الترطيب" (Hydration) لربط كتل الإعدادات وحقول الإدخال بمعالجات الكتابة ومحركات الحفظ على ملف البيئة وتطبيق تعديلات منافذ البث والحماية الفورية.

---

## ٢. تفكيك وتحليل محتويات ملف العرض الساكن (`cp.html`)

يحتوي ملف `cp.html` المترجم على البنية الفنية والرسومية التالية:

1. **الترويسات والأصول الساكنة (Headers & Layout styles):**
   * تنسيقات الـ CSS الرئيسية المجمعة لمنح لوحة التحكم ومفاتيح التبديل مظهراً عصرياً وجذاباً (Modern Indigo Design Style).
   * وسوم منع التخزين المؤقت لضمان عدم بقاء كلمات المرور والمفاتيح السرية الحساسة بذاكرة المتصفح.

2. **DOM Tree للوحة التحكم:**
   * التبويبات الفنية مسبقة الرندر للخيارات العامة، الشبكة، التراخيص، والأمان المتقدم.
   * حقول تعيين منافذ البث (HTTP Port) ومنافذ الـ RTMP.
   * حقل إدخال مفتاح تشفير التوثيق (JWT Secret) مسبق التنسيق والحماية.

3. **حقنة بيانات المزامنة (Dehydrated React State JSON):**
   * نصوص جافا سكريبت التفاعلية المضمنة لضمان استقرار وتوافقية المتصفح مع خيارات الكتابة والرفع على السيرفر.

---

## ٣. معوقات ملف لوحة التحكم الساكن في بيئات Docker وسيرفرات QNAP NAS

عند تشغيل مباشر ستريم في حاويات Docker على سيرفرات **QNAP NAS** تبرز معوقات متعلقة بالملفات الساكنة للوحة التحكم:

1. **فشل تحديث وعرض القيم الحقيقية لمتغيرات البيئة (Stale Environment values):**
   بما أن ملف HTML مسبق الرندر ساكن، فقد لا تظهر التعديلات الأخيرة لملف التكوين `.env` بالتبويبات بشكل تلقائي إلا بعد إعادة تحميل أو ترطيب تفاعلي مستمر.
   * **الحل الفني والترحيل السليم:** إسناد مهمة قراءة قيم الإعدادات لـ API ديناميكي خلفي يتم استدعاؤه فورياً بالمتصفح لتجاوز أي قيم ساكنة قديمة مخزنة بالـ HTML Shell.
2. **عقبة الكتابة وقفل ملف البيئة تحت نظام ملفات كيوناب:**
   عند تعديل وحفظ منفذ من منافذ لوحة التحكم، قد تظهر رسالة فشل الكتابة بسبب قيود حماية الـ Volumes الممررة للحاوية.
   * **توصية الترحيل:** ضبط أذونات وصلاحيات الحاوية وتأمين المجلدات ومطابقة الـ UID لمدير السيرفر للسماح بالكتابة بسلاسة وبلا قيود.

## ٤. دليل كسر القيود وضمان إتاحة لوحة التحكم العامة للأبد

لمنع حظر صمامات لوحة التحكم وإعدادات النظام والنواة بالمنصة للأبد:

* **تغذية متغيرات التخطي البيئية لـ Docker container:**
  نمرر متغيرات بيئة التخطي لـ Docker لتجاوز قيود التراخيص والفترات التجريبية المدمجة بالصفحة:
  ```env
  BYPASS_EXPIRE_CHECK=true
  TRIAL_MODE=false
  LICENSE_KEY="MUBASHER_STREAM_ENTERPRISE_UNLIMITED_PRO_PERPETUAL_2026_BYPASS_OK"
  ```
